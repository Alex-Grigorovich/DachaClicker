# Clicker Design Doc

## Обзор
Проект уже представляет собой рабочий прототип кликера на Cocos Creator с выбором овощей для ячеек, базовой экономикой, разблокировкой части контента за валюту и UI-адаптацией под экран. Основной игровой цикл сейчас выглядит так:

1. Игрок кликает по выращенному овощу.
2. `VegClickMoney` начисляет валюту через `MoneyManager`.
3. Игрок открывает `VegetableList` из свободной ячейки.
4. При необходимости оплачивает разблокировку строки меню.
5. Выбирает культуру, и префаб помещается в `Content` целевой ячейки.

## Текущее состояние систем

### 1. Экономика
- `MoneyManager.ts` хранит текущий баланс в `Label`, умеет начислять и списывать деньги.
- Баланс при запуске сбрасывается в `0`.
- `VegClickMoney` и `VegetableMenuHandler` уже используют `MoneyManager` как основной источник денег.

### 2. Основной кликер-цикл
- `VegClickMoney.ts` ищет ноды `VegClick` и вешает на них клик.
- За клик выдаётся `addPerClick`.
- Реализованы:
  - локальный кулдаун;
  - анимация подпрыгивания;
  - полёт монетки к `Moneybar`;
  - синхронизация цены в `moneyCountLabel`.

### 3. Слоты и выбор культуры
- `SlotMenuHandler.ts` открывает `VegetableList` из ячейки.
- Текущая ячейка передаётся в `VegetableMenuHandler`.
- После выбора префаб добавляется в `Content`.
- Повторное открытие меню для уже занятой ячейки запрещено.

### 4. Меню овощей
- `VegetableMenuHandler.ts` поддерживает 4 культуры:
  - carrot;
  - cabbage;
  - tomato;
  - chili.
- Для строк `CellListCabbage`, `CellListTomato`, `CellListChiliPepper` уже реализованы блоки разблокировки:
  - цена берётся из `Label` на `cellListBlock...`;
  - если денег хватает, сумма списывается;
  - блок скрывается;
  - кнопка строки становится активной.

### 5. Блокировка ячеек
- `CellLockHandler.ts` умеет:
  - показывать замок;
  - блокировать нажатие на саму ячейку;
  - проверять цену разблокировки по `labelLock`;
  - включать режим `lockWithoutPrice`.
- Важное ограничение: сейчас эта система не интегрирована нормально с `MoneyManager`.

### 6. UI и сервисные скрипты
- `ResolutionAdapter.ts` масштабирует игровое поле и боковые кнопки под размер окна.
- `FontLoader.ts` грузит шрифт из `resources`.
- `GlobalCooldown.ts` реализован как отдельная система, но пока не подключён к остальному циклу.
- `MoneyBar.ts` работает с `EditBox`, но логически дублирует часть функций `MoneyManager`.

## Что уже готово
- Базовый игровой цикл клика.
- Начисление валюты.
- Система меню выбора овощей.
- Установка овоща в слот.
- Частичная прогрессия через платную разблокировку меню.
- Отдельная блокировка ячеек замком.
- UI-адаптация и кастомный шрифт.

## Что ещё не доведено до целостной системы

### 1. Разрозненная логика денег
Сейчас в проекте есть два способа работы с деньгами:
- `MoneyManager` через `Label`;
- `MoneyBar` через `EditBox`.

Это создаёт риск рассинхрона и усложняет дальнейшее развитие.

### 2. Незавершённая механика замка ячейки
`CellLockHandler` проверяет баланс через отдельный `moneyTextNode`, а не через `MoneyManager`.

Кроме того, списание денег фактически не завершено:
- `deductMoney()` пока только логирует;
- списание в обработчике разблокировки не включено как полноценная игровая логика.

### 3. Неиспользуемый глобальный кулдаун
`GlobalCooldown.ts` выглядит как заготовка, но не участвует в текущем цикле кликов.

### 4. Хрупкие зависимости от структуры сцены
Есть жёсткие пути и имена нод:
- `Canvas/UI/VegetableList`;
- `ButtonClose/ButtonClose`;
- `Content`;
- `VegClick`;
- `cellListBlock...`.

Это ускоряет прототипирование, но затрудняет масштабирование.

## Основные риски
- Рассинхрон между разными UI денег.
- Ломкость логики при изменении иерархии сцены.
- Рост количества исключений и ручных проверок при добавлении новых овощей.
- Повторение логики клика и разблокировки в разных местах.

## Что логично добавить дальше

### Высокий приоритет
1. Сделать `MoneyManager` единственным источником правды для денег.
2. Подключить `CellLockHandler` к `MoneyManager` и завершить списание при разблокировке.
3. Сохранение прогресса:
   - баланс;
   - разблокированные пункты меню;
   - занятые ячейки;
   - открытые/закрытые замки.
4. Упростить связи со сценой через `@property`, где это возможно, вместо жёстких путей.

### Средний приоритет
1. Подключить или удалить `GlobalCooldown`.
2. Добавить фидбек на нехватку денег:
   - звук;
   - визуальный отказ;
   - подсветка цены.
3. Сделать форматирование больших чисел (`1 000`, `10K`, `1.2M`).
4. Добавить состояние выбранной/посаженной культуры как данных, а не только через наличие дочернего префаба.

### Следующий этап развития игры
1. Апгрейды дохода за клик.
2. Пассивный доход или автосбор.
3. Новые культуры и уровни прогрессии.
4. Магазин улучшений.
5. Баланс экономики и стоимость разблокировок.

## Рекомендуемая целевая архитектура

### Ядро
- `MoneyManager` — единая экономика.
- `ProgressManager` — сохранение/загрузка прогресса.
- `UnlockManager` — статусы разблокировок.
- `PlantManager` или данные слота — что посажено в каждой ячейке.

### Игровые компоненты
- `VegClickMoney` — только логика клика по овощу.
- `SlotMenuHandler` — только открытие/закрытие UI меню.
- `VegetableMenuHandler` — только выбор культуры и разблокировки меню.
- `CellLockHandler` — только блокировка конкретной ячейки.

### UI
- один компонент отображения денег;
- единые правила показа ошибок и состояний;
- минимизация поиска по строковым путям.

## Итог
Сейчас проект находится на стадии хорошего игрового прототипа: фундамент кликера уже собран, меню посадки и часть прогрессии работают, но архитектура ещё не приведена к единому игровому состоянию. Главная задача следующего этапа — не столько добавление новых фич, сколько выравнивание экономики, сохранения и логики разблокировок в одну связную систему.
